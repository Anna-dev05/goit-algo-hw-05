## Алгоритми пошуку та порівняння ефективності
### Завдання 1 — HashTable
Додано метод `delete` для хеш-таблиці, реалізованої з використанням
окремих ланцюжків (separate chaining).
Метод видаляє пару ключ–значення та повертає `True`, якщо ключ було знайдено,
або `False` — якщо ключ відсутній.

---

### Завдання 2 — Двійковий пошук
Реалізовано двійковий пошук для відсортованого масиву дробових чисел.
Функція повертає кортеж:
- кількість ітерацій пошуку;
- upper bound — найменший елемент, який є більшим або рівним заданому значенню
  (або `None`, якщо такого елемента не існує).
  
---

### Завдання 3 — Порівняння алгоритмів пошуку підрядка

Порівняно три алгоритми:
- **Боєра–Мура**
- **Кнута–Морріса–Пратта (KMP)**
- **Рабіна–Карпа**

Для двох текстових файлів проведено заміри часу виконання
за допомогою модуля `timeit` для:
- підрядка, який існує в тексті;
- вигаданого підрядка, який у тексті відсутній.

#### Результати
Для обох текстів і обох типів підрядків найшвидшим алгоритмом
виявився **Боєр–Мур**.
Алгоритм **KMP** показав середній результат,
а **Рабін–Карп** був найповільнішим у даному тестуванні.

Це підтверджує, що алгоритми зі “стрибками” по тексту
часто працюють швидше на реальних даних.
